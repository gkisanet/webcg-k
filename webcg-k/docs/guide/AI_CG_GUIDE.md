# AI CG Graphics Generation Guide

> Guides and references detailing how to generate broadcast character generator (CG) graphics using AI pipelines and describes the underlying element data structures in WebCG-K.

---

## 1. Graphics Generation Workflow

### 1.1 UI-Driven Creation (OverlayCreationWizard)

Clicking **"Create Overlay"** in the Dashboard ➔ Overlays tab launches a 4-step creation wizard:

| Step | Step Name | Purpose |
|------|-----------|---------|
| 1 | Grid Selection | Choose screen division template (e.g. 4 quads, lower band + locator) |
| 2 | Zone Selection | Select target zones to map overlays (multi-select supported) |
| 3 | AI Prompting | Describe design requests in natural language + link data sources |
| 4 | Variation Selection | Preview 4 candidate designs generated by Gemini and save the preferred variation |

### 1.2 Programmatic Generation

To generate graphics programmatically without visual wizard triggers, consume the **`generateCgVariations()` function**:

```typescript
import { generateCgVariations } from "@/services/aiCgService";

const variations = await generateCgVariations(
  "Create weather lower-third showing Seoul's temperature, humidity, and micro-dust index.",
  { x: 0, y: 810, width: 1920, height: 270 },    // Lower Third Coordinates
  { temperature: 23, humidity: 45, dust: "Good" }  // Variable Data Bindings
);
```

**Parameters:**
- `prompt` (string) — Natural language design descriptions.
- `bounds` (ZoneBounds) — Target overlay coordinates (in pixels).
- `dataContext` (optional) — Variables map object for live data bindings.

**Output:** `CgVariation[]` — Array containing up to 4 styling variations.

## 2. Writing Effective Prompts

### 2.1 Base Structure

```
[CG Format Type] + [Information Fields] + [Visual Styling Directives]
```

### 2.2 Prompt Examples

| Target Format | Prompt Example | Strengths |
|------|---------|------|
| **News Lower Third** | "News lower-third displaying reporter name and affiliation. Left-aligned blue vertical accent bar, large bold font for name, and small muted font for 'Political Desk Reporter'." | Explicitly declares layouts, color values, and typographic hierarchy. |
| **Breaking News Ticker** | "Red breaking news banner spanning the top header. 'BREAKING' text in white bold, with story content text aligned underneath." | Declares placement zone, colors, and font scaling structures. |
| **Weather Forecast** | "Weather forecast overlay. Large temperature index displays on the left, humidity and micro-dust values stack on the right. Dark translucent gradient sheet background." | Declares layout split ratios and modern visual layers. |
| **Sports Scoreboard** | "Football match scoreboard. Left team vs right team, center scores, and elapsed play match timer aligned below." | Emphasizes symmetrical structures. |
| **Channel Watermark** | "Channel logo in the top right reading 'KBS'. Translucent gray tone, small footprint." | Coordinates placement and opacity thresholds. |

### 2.3 Prompting Tips

1. **Declare exact regions** — *"on the left", "at the header", "in the lower 1/4 zone"*
2. **Describe layering hierarchy** — *"translucent strip overlapping a dark background, with text on top"*
3. **Specify color palettes** — *"blue and white tokens", "light fonts on high-contrast backgrounds"*
4. **Be concrete, avoid vague labels** — ❌ *"Beautiful lower-third"* ➔ ✅ *"Gradient overlay bar with neon glowing accents"*
5. **Data binding cues** — Register dynamic values in the `dataContext` and reference key terms in your prompt.

### 2.4 Bad Prompt Examples

| Prompt Example | Core Pitfalls |
|---------|------|
| "Make a cool graphic." | Vague — does not declare layouts, targets, or styles. |
| "A graphic with awesome motion slides." | Playout engines process static layouts — motions are not generated. |
| "A thumbnail for my YouTube channel." | Generates off-air media assets — clashes with the playout system's parameters. |

## 3. GraphicElement Data Structure Reference

### 3.1 Core Interface

All visual elements returned by the generator are mapped within a flat `GraphicElement[]` array:

```typescript
interface GraphicElement {
  // ─── Required Properties ─────────────────────
  id: string;              // Unique identifier (e.g., "el-1", "bg-rect")
  type: "rect" | "ellipse" | "text" | "image" | "group";
  name: string;            // Descriptive label (e.g., "Background Rectangle")
  x: number;               // X coordinate (in pixels, from canvas top-left)
  y: number;               // Y coordinate
  width: number;           // Width (in pixels)
  height: number;          // Height (in pixels)
  rotation: number;        // Angle rotation (in degrees)
  opacity: number;         // Element transparency (0.0 to 1.0)
  visible: boolean;        // Playout visibility toggle
  zIndex: number;          // Drawing order (0 represents background)
  parentId: string | null; // ID of parent group container (null for top-level)
  
  // ─── Styling Properties ──────────────────────
  fill?: Fill;              // Element fill definitions (solid / gradient / none)
  stroke?: Stroke;          // Border parameters
  borderRadius?: number;    // Corner rounding radius (in pixels)
  customCSS?: string;       // Custom CSS overrides (advanced)
  
  // ─── Text Properties ─────────────────────────
  content?: string;         // Plain text contents
  fontFamily?: string;      // Font-family name (e.g., "Pretendard")
  fontSize?: number;        // Font size (in pixels)
  fontWeight?: number;      // Typography weight (400 for regular, 700 for bold)
  lineHeight?: number;      // Text height multiplier (e.g., 1.2)
  letterSpacing?: number;   // Character spacing margins (in pixels)
  textAlign?: "left" | "center" | "right" | "justify";
  verticalAlign?: "top" | "middle" | "bottom";
  
  // ─── Image Properties ────────────────────────
  src?: string;             // Asset absolute URL
  objectFit?: "cover" | "contain" | "fill";
  
  // ─── Group Properties ────────────────────────
  children?: string[];      // Array containing child element IDs
  
  // ─── Visual Effects ──────────────────────────
  shadowEnabled?: boolean;
  shadowColor?: string;
  shadowOffsetX?: number;
  shadowOffsetY?: number;
  shadowBlur?: number;
}
```

### 3.2 Fill Options

```typescript
interface Fill {
  type?: "solid" | "linear" | "radial" | "none";
  color?: string;           // HEX code (e.g., "#1a1a2e")
  opacity?: number;         // 0.0 to 1.0
  // Gradient parameters
  gradientAngle?: number;   // Gradient angle (0 to 360)
  gradientStops?: Array<{
    offset: number;         // Offset percentage (0 to 100)
    color: string;
    opacity?: number;
  }>;
}
```

### 3.3 Stroke Options

```typescript
interface Stroke {
  color: string;
  width: number;
  style: "solid" | "dashed" | "dotted";
  opacity?: number;
}
```

### 3.4 Guidelines by Element Type

| Element Type | Use Case | Crucial Parameters | Visual Example |
|---|---|---|---|
| `rect` | Background frames, visual lines, cards | `fill`, `borderRadius` | Lower third backgrounds, card boxes |
| `ellipse` | Circle badges, avatar frames, logo backs | `fill` | Channel logo backing rings |
| `text` | Headers, names, metrics numbers | `content`, `fontSize`, `fontFamily` | Reporter names, clock feeds |
| `image` | Logos, watermarks, photos | `src`, `objectFit` | Network identification bug |
| `group` | Multi-element wrappers | `children` | Associated Name + Role container |

## 5. Under-the-Hood Processing Pipeline

### 5.1 Pipeline Flowchart

```
Operator Prompt
    ↓
generateCgVariations(prompt, bounds, data?)
    ↓
System Prompts + Operator Prompt ➔ LLM API (Gemini/DeepSeek/Groq)
    ↓
LLM Output Response (JSON string)
    ↓
parseElementsFromResponse() ➔ Standardizes GraphicElement[] structures
    ↓
CgVariation[] (4 designs) ➔ Rendered inside Wizard selections UI
    ↓
Save preferred choice in "overlay_templates" DB table
    ↓
OverlayPlayoutLayer ➔ Rendered as vector SVGs by GraphicPreviewRenderer
```

### 5.2 System Prompt Protocol

Rules governing the system instructions passed to the AI models:

1. **Output pure JSON only** — No markdown wrappers, commentary, or text prefixes.
2. **Adhere strictly to `GraphicElement` models** — Leverage typings detailed in Section 3.
3. **Korean text contents** — Variables in the "content" property should default to Korean.
4. **Professional quality standards** — Styling tokens must mimic premium television broadcast designs.
5. **High-contrast, high-visibility colors** — Select visual tokens optimized for legibility against dynamic underlying live video feeds.

### 5.3 Administrative Settings

Operator Admin panels (Dashboard ➔ Admin ➔ AI Settings) manage model configurations:
- **Model Toggles**: Switch between Gemini 2.5, DeepSeek, Groq, GitHub, and OpenRouter endpoints.
- **System Prompt Overrides**: Tweak the underlying graphic system prompt to adjust design biases globally.
- **Credential PURGE**: Purge and manage API keys saved in DB records or environmental variables.
- **Generation Parameters**: Configure `temperature`, `maxOutputTokens`, `topP`, and `topK`.

## 6. Constraints & Roadmap

### Present Constraints

| Constraint | Details |
|---|---|
| **Static layouts only** | Generates shapes and text arrangements without animations or motions. |
| **SVG Renderer** | Does not support raw CSS inline animations (limited to standard Fade In/Out properties). |
| **Incomplete data binding** | Does not auto-repaint overlays when background data feeds update. |
| **Font constraints** | Custom typography must be pre-registered within the Font Registry. |
